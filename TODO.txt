[3] Freeze the timer after the player confirms or discards their action
------------------------------------------------------------------------------
Context:
  - In table.component.ts, `startTimer()` runs a `setInterval` that ticks every
    second. It is only cleared on `onTimeUp()` or when a new turn starts via
    `gameStateService.newTurn.subscribe(() => startTimer())`.
  - `onConfirmOrDiscard()` (table.component.ts) calls `gameStateService.playAction()`
    which sends the move to the server, but does NOT stop or freeze the timer.
  - The countdown sound (`playCountdownTick`) keeps playing even after the player
    has already sent their action (lines 253–255).
  - The server controls the actual turn timeout independently; the client timer
    is only a local visual aid.

Tasks:
  a. In `onConfirmOrDiscard()`, after calling `gameStateService.playAction()`:
       - Call `clearTimer()` to stop the setInterval.
       - Leave `timeLeft` signal at its current frozen value (do NOT reset to 0).
       - This visually freezes the SVG arc at the current position.
  b. Ensure `playCountdownTick()` is no longer called after the action is sent
     (clearing the interval is sufficient since the sound is triggered inside
     the interval callback).
  c. The timer will restart naturally when the backend sends the next turn and
     `gameStateService.newTurn` emits, which already calls `startTimer()`.
  d. No server-side changes are needed.

[6] Redesign the player-info panels and tie the timer to the current-turn player
------------------------------------------------------------------------------
Context:
  - The `.player-card` is 5×square-size wide but only ~0.85×square-size tall
    (one grid row). Its current single-row layout (avatar + name + 5 card dots)

      • The `.disconnected-badge` is `position: absolute` on the right and
        overlaps `.player-info` content rather than having its own space.
      • There is no room for an individual turn timer.
  - The SVG ring timer in table.component.ts (`timerCircumference`,
    `timerDashOffset`, `timerColor`) always displays when a turn is active,
    regardless of whose turn it is.
  - `gameStateService.isMyTurn()` returns true only for the local player's turn.
    `displayedGameData()?.gameState.currentTurn` holds the active player's color.
  - board.component.html already binds `[class.active-turn]` on `.player-card`.
  - The design must remain aesthetic and harmonious with the existing glassmorphism
    theme (dark background, color accents per player, rounded cards).

Tasks:
  a. Redesign `.player-card` layout in board.component.html / board.component.scss:
       - Keep the left `avatar-ring` (can shrink slightly: 50% instead of 60% of
         square-size) to free up horizontal space.
       - Change `.player-info` to `flex-direction: column` with two mini-rows:
           • Top row: `.player-name` — give it `flex: 1; min-width: 0` so it can
             shrink naturally. Use a slightly smaller font
             (e.g. `calc(var(--square-size) * 0.32)`) and keep `text-overflow:
             ellipsis` only as a last resort — with column layout and a proper
             min-width it should rarely truncate.
