### 1. Fix 7-card split movement preview

**File:** `frontend/src/app/home/components/board/board.component.ts` → `previewInfo` computed signal

**Context:** When a player plays the 7 card, they must distribute 7 steps between two of their own marbles. The selection flow is:
  1. Player selects a card (7)
  2. Player selects the step count to give to the first marble (e.g., 3)
  3. Player clicks the first marble → marble1 is now selected (`selectedMarblePosition`)
  4. Player clicks the second marble → marble2 is now selected (`selectedSplit7MarblePosition`)

**Expected preview behavior:**
  - **Phase 3 (marble1 selected, marble2 not yet selected):**
    - Always show marble1's path preview (for `steps1` steps), regardless of where the mouse is
    - Additionally, when hovering over a valid candidate for marble2 (a different own marble), also show that marble's path preview (for `7 - steps1` steps) on top of marble1's preview
  - **Phase 4 (both marble1 and marble2 selected):**
    - Always show both marble1's path preview (for `steps1` steps) AND marble2's path preview (for `7 - steps1` steps) simultaneously

**Current bug:** The second marble's preview is not shown correctly. Specifically:
  - In phase 3, when hovering over a candidate marble2, marble1's preview may disappear
  - In phase 4, only one marble's path may be shown rather than both simultaneously

**Implementation hint:** In `previewInfo`, marble1's preview should always be included in the result when `selectedMarblePosition` is set and the card is 7. Do not let `hoveredMarble` override marble1's preview — instead, always include marble1 and layer the hovered/selected marble2 preview on top.
