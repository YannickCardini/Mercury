TUTORIAL OVERLAY — REFINEMENTS
==============================

Context
-------
The tutorial overlay (frontend/src/app/game/components/tutorial-overlay/) shows
contextual hints on the local player's turn. Each hint is a text "pill" with a
pointer arrow, plus one or more yellow highlight rings around the target.
There are 4 hints, driven by `hint()` in tutorial-overlay.component.ts:
  - 'card'    → "Select a card from your hand"   (anchor: 'hand')
  - 'marble'  → "Select a marble to move"        (anchor: 'board')
  - 'confirm' → "Tap the button to confirm"      (anchor: 'confirm')
  - 'discard' → "No playable cards — tap to discard" (anchor: 'confirm')

What already works well — DO NOT CHANGE
---------------------------------------
- The 'confirm' / 'discard' hint: pill + highlight on the confirm/discard
  button (.action-btn, anchor 'confirm') is good as-is.
- The 'marble' hint's highlight: the circular yellow ring around the
  selectable marble (.highlight-board) is good as-is.

The text pill + arrow for the 'card' and 'marble' hints look bad and need to
be reworked. The two tasks below cover that.


TASK 1 — Reposition the 'marble' hint pill next to the player's home corner
---------------------------------------------------------------------------
Problem:
  The 'marble' hint currently places the pill at the union rect of all
  `.marble-selectable` elements, above or below the board centre
  (see `recompute()` -> `anchor === 'board'` branch). This placement looks bad.

Key fact:
  The 'marble' hint only ever appears on the player's very first action
  (`hasConfirmed` is still false). On the first action the only selectable
  marble is one marble sitting in the local player's home position. So the
  hint always points at a single marble in the player's home corner.

Required change:
  - In the 'board' branch of `recompute()`, position the pill + arrow next to
    the local player's home position instead of the board-centre-relative
    union placement.
  - Identify the player's home squares from the board: home squares render
    with class `home home-${color}` (see board.component.ts ~line 710,
    `getSquareClass`). Use `gameStateService.myPlayerColor()` to pick the
    color, e.g. query `.home-${myColor}`.
  - Place the pill just outside that home cluster, with the arrow pointing
    toward the marble. Keep the existing viewport clamping
    (`clampPillIntoViewport`).
  - Keep the existing circular highlight ring on the selectable marble
    unchanged.


TASK 2 — Rework the 'card' hint: highlight style + pill placement
-----------------------------------------------------------------
Problem:
  a) The 'card' hint highlight currently uses `.highlight-hand` — a soft,
     blurred radial-gradient glow (see tutorial-overlay.component.scss). This
     design looks bad.
  b) The pill placement above the card fan needs to be improved.

Required changes:
  a) Replace the `.highlight-hand` blurred-glow style with a crisp yellow
     border ring, matching the style already used for marbles/buttons
     (`.highlight-board` / `.highlight-confirm`: `box-shadow` ring +
     `highlightPulse` animation). Each `.playable-card` should get its own
     ring. Note the card rings must still follow each card's fan rotation —
     `orientedBox()` already computes per-card rotation; keep that working.
  b) Improve the pill + arrow placement for the 'card' hint so it sits
     cleanly relative to the highlighted card fan (currently centred above
     the union rect of `.playable-card` elements). Keep viewport clamping.


Out of scope
------------
- Do not change the 'confirm' / 'discard' hint.
- Do not change the hint state machine / trigger logic in `hint()`.
- Do not change when hints retire (`hasConfirmed` / `hasDiscarded`).
