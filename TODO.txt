================================================================================
FEATURE — Chat / emojis animés pendant une partie
================================================================================

Objectif
--------
Permettre aux joueurs d'interagir de façon conviviale pendant la partie
(messages courts ou emojis animés type "réactions") sans encombrer l'interface
de jeu déjà dense sur mobile.

Contraintes
-----------
- La majorité des joueurs sont sur mobile, écran réduit, interface déjà bien
  remplie (plateau + main + timer + footer).
- Aucune régression sur le gameplay existant.
- Ergonomie pouce-friendly.

Piste à explorer (à valider)
----------------------------
Ajouter un système de tabs (ou de switch) AU-DESSUS du `TableComponent`
permettant de basculer entre :
  - Vue "cartes" (actuelle, par défaut),
  - Vue "chat" (nouvelle).
Le switch ne doit pas dégrader la lisibilité du plateau.

Travail préliminaire à faire (pas d'implémentation tant que le design n'est
pas validé)
--------------------------------------------------------------------------
a) Lister les options de design possibles :
     1. Tabs au-dessus du TableComponent (cartes ↔ chat), comme suggéré.
     2. Bulle "réactions" flottante en coin (emojis animés uniquement, pas de
        texte libre).
     3. Tiroir latéral / bottom-sheet ouvert via une icône (icône à intégrer
        dans `.action-menu`).
     4. Combinaison (emojis flottants tap-rapide + chat texte caché dans un
        tiroir).
   Pour chaque option, lister avantages / inconvénients UX et impact sur le
   layout actuel.

b) Décider ensemble (utilisateur + IA) :
     - Texte libre OU emojis prédéfinis OU les deux ?
     - Modération / signalement nécessaire ?
     - Historique persistant ou éphémère à la fin de la partie ?

c) Définir le contrat WebSocket :
     - Nouveau message client→serveur (p. ex.
       `ChatMessage` : `{ type: 'chat', content: string }` ou
       `EmojiReaction` : `{ type: 'reaction', emoji: string }`).
     - Nouveau message serveur→clients (broadcast à la room) avec horodatage
       et auteur (couleur + nom).
     - À placer dans packages/shared/src/types.ts.

d) NE PAS coder l'implémentation tant que les points (a), (b), (c) ne sont
   pas tranchés avec l'utilisateur.
