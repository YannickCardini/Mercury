## TASK 2 — Fix visual misalignment of the discard pile on the board

Context:
- The discard pile is rendered in `board.component.html:86-100` inside a
  `.discard-zone` div that is absolutely positioned at `left: 50%; top: 50%;`
  with `transform: translate(-50%, -50%)` (SCSS lines ~698-709).
- Each stacked card is offset with
  `transform: translate(calc(index * -3px), calc(index * 3px)) rotate(...)`,
  making each card shift left relative to the previous one.
- Because the stack grows to the left, the top card ends up left of the
  intended center, making the entire pile look off-center on the board.

Steps:
1. In `board.component.scss`, adjust the `.discard-zone` centering transform to
   compensate for the leftward drift of the stack. For a maximum of 5 visible
   cards offset by -3px each, the total leftward shift is up to -12px (4 gaps).
   Add `translateX(6px)` to the `.discard-zone` transform to re-center the
   visual midpoint of the pile:
     `transform: translate(calc(-50% + 6px), -50%);`
   Tune the pixel value if needed so the pile appears centered between the draw
   deck and the board edges.
2. Alternatively, change the card stacking direction so cards cascade to the
   right (positive X) and upward (negative Y) instead, which keeps the
   top/most-recent card roughly at the center point.
3. Do not remove the cascade effect — only fix the centering.


## TASK 3 — Rename unauthenticated players as "Guest #1" through "Guest #4"

Context:
- When a player is not logged in, the frontend currently calls `generateGuestName()`
  (defined in `frontend/src/app/shared/guest-name.ts`) which generates a random
  name like "SwiftFox42". The desired behaviour is to assign a deterministic
  name "Guest #X" where X reflects the order in which the player joined the game
  (1st joiner → "Guest #1", …, 4th joiner → "Guest #4").
- Player colors are assigned in join order by the backend
  (`backend/src/session/session-manager.ts:joinRoom()`). The color-to-slot
  mapping is: RED = slot 1, GREEN = slot 2, BLUE = slot 3, ORANGE = slot 4
  (based on the `MarbleColor` enum in `packages/shared/src/types.ts`).
- `generateGuestName()` is called in `frontend/src/app/home/home.page.ts`
  at lines ~321 and ~469 when `user` is null.

Steps:
1. In `frontend/src/app/home/home.page.ts`, replace calls to `generateGuestName()`
   with a helper that derives the guest number from the player's assigned color slot:
     - RED   → "Guest #1"
     - GREEN → "Guest #2"
     - BLUE  → "Guest #3"
     - ORANGE → "Guest #4"
   The player's color is available in the game config at the point these calls
   are made. Pass it to the new helper instead of using the random generator.
2. Update (or replace) the `generateGuestName()` function in
   `frontend/src/app/shared/guest-name.ts` to accept a `MarbleColor` parameter
   and return the corresponding "Guest #X" string. Remove the adjective/animal/
   number random logic entirely since it will no longer be used.
3. Ensure the generated name is sent to the backend in the `JoinGameMessage`
   (`packages/shared/src/types.ts:315-319`) so other players see the correct
   guest name in the board's player corners.
4. No backend changes are required — the backend stores whatever name the client
   sends and does not generate guest names itself.
